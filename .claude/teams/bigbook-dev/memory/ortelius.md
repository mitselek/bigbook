[GOTCHA] **V8 coverage and `noUncheckedIndexedAccess` interact badly.** Every regex capture-group fallback (`match[1] ?? ''`) that exists only to satisfy the strict tsconfig is a dead branch under v8 — the outer regex match guarantees the inner captures are defined, but v8's branch counter cannot see that guarantee. Three alternatives documented in my P1.7 CYCLE_COMPLETE quality notes, in order of preference for future modules under `src/lib/content/`:

1. **String slicing over regex groups** where feasible — `content.slice(start, end)` returns `string`, never `string | undefined`. Suitable when boundaries are known by position.
2. **Typed narrowing helpers** like `assertDefined(x, 'match[1]')` or `nonNull<T>(x: T | undefined): T` that throws. Exchanges the dead branch for a thrown-error branch that IS reachable in principle, which v8 typically doesn't flag.
3. **Destructure-and-check consolidation**: `const [, a, b] = match` then `if (a === undefined || b === undefined) throw`. Same branch cost but consolidates to one place per match.

For `parse.ts` the v8-ignore pattern is fine — small module, explicit localized annotations. For `validate.ts` and `diff.ts`, if the plan's impl sketch uses regex groups and strict indexing, either flag it to Plantin as structural guidance *before* Granjon starts the GREEN cycle, or coach Granjon toward pattern 1/2/3 during a PURPLE refactor. Catching this at the plan review stage is cheaper than the P1.7 escalation was.

[GOTCHA] **The `forEach(line, index)` conversion is the right call for index-tracking loops.** Granjon flagged converting `for-of` → `forEach` in P1.6 to get the `index` for 1-based error line numbers. The alternatives — manual counter variable with `for-of`, or `for (let i = 0; i < lines.length; i++)` — both add mutable state for no benefit. `forEach` with `return` (= `continue`) and `throw` (propagates immediately, identical to `for-of`) has equivalent semantics and is more concise. The closure over `currentId`/`currentLines` still works because `flush` captures by reference. Don't reject this conversion in future GREEN handoffs.

[GOTCHA] **Line numbers shift when annotations add lines.** Plantin's v8-ignore directive referenced lines 35, 36, 51, 92 in the pre-annotation source. After Granjon added four `/* v8 ignore next */` comment lines, the actual annotated target statements landed at lines 36, 38, 54, 96. When verifying placement, map by *target statement identity* (`match[1] ?? ''`, `match[2] ?? ''`, `key === undefined || val === undefined`, `directive[1] ?? null`), not by literal line number. The identity is the contract; the line number is a transient pointer.
