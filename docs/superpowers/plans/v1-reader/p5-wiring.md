# P5 — Runtime Wiring

**Phase goal:** Wire everything together — `ChapterSection.svelte` island (fetch → parse → diff → render), IntersectionObserver preload, TopBarClient reactive title, visibility-change refresh, IndexedDB persistence, return-visit scroll. After this phase, the reader is **alive**: scrolling triggers lazy fetch, content replaces skeletons, marginalia shows diffs, the top bar title updates.

**Execution mode:** Inline (Plantin). This is integration wiring — the pure modules and components are already tested from P0–P4.

**Files:**

- Create: `src/components/ChapterSection.svelte`
- Create: `src/components/TopBarClient.svelte`
- Modify: `src/pages/index.astro` (replace static skeleton sections with ChapterSection islands)
- Modify: `src/components/TopBar.astro` (embed TopBarClient island in center zone)

## Task P5.1 — ChapterSection.svelte scaffold

- [ ] Create `src/components/ChapterSection.svelte` — a Svelte 5 island that manages the fetch/render lifecycle for one chapter.

**Props:**

```ts
type ChapterSectionProps = {
  slug: string
  paraIds: readonly string[]
  titleEn: string
  titleEt: string
}
```

**Behavior:**

1. Initially renders blank skeleton rows (same markup as P2.3's static skeleton, but generated by Svelte).
2. Exposes a `load()` method (or reactive trigger) that fires the three parallel fetches via `fetchEn`, `fetchBaselineEt`, `fetchCurrentEt` from `src/lib/content/fetch.ts`.
3. On fetch success: parse each via `parse()`, compute `diff(currentEt, baselineEt)`, replace skeletons with `ParagraphRow` components.
4. On fetch error: render an error row with a retry button.

Scaffold the component with skeleton rendering only (no fetch wiring yet).

- [ ] Verify: `npm run build` succeeds, `npm run typecheck` clean.
- [ ] **Commit:** `feat(ChapterSection): P5.1 — scaffold with skeleton rendering`

## Task P5.2 — ChapterSection fetch → render pipeline

- [ ] Wire the fetch pipeline into `ChapterSection.svelte`:

1. Import `fetchEn`, `fetchBaselineEt`, `fetchCurrentEt` from `src/lib/content/fetch`.
2. Import `parse` from `src/lib/content/parse`.
3. Import `diffCurrentVsBaseline` from `src/lib/content/diff`.
4. On `load()` trigger: set state to `'loading'`, fire three parallel fetches, on success parse all three, compute diff, set state to `'loaded'` with the parsed data.
5. When loaded, iterate `paraIds` and render a `ParagraphRow` for each, passing the EN text, ET text, diff status, and baseline ET text (when diverged) as props.

- [ ] Verify: manually test by temporarily auto-loading one chapter in `index.astro`. Content should render in place of the skeleton.
- [ ] **Commit:** `feat(ChapterSection): P5.2 — fetch → parse → diff → render pipeline`

## Task P5.3 — Wire IntersectionObserver preload into index.astro

- [ ] Modify `src/pages/index.astro`:

1. Replace static skeleton `<section>` elements with `<ChapterSection client:visible>` islands (Astro's `client:visible` uses IntersectionObserver under the hood).
2. Each `ChapterSection` receives its manifest data as props.
3. Remove the static skeleton CSS (now handled by ChapterSection).

**Alternative if `client:visible` preload margin is insufficient:** Use `client:idle` for all chapters (hydrate immediately) and let each `ChapterSection` create its own preload observer using `createPreloadObserver` from `scroll-anchor.ts`. The observer callback triggers `load()`.

Choose whichever approach produces the correct behavior during implementation. The spec requires content to preload when the chapter section is ~1.5 viewports away, not when it's visible.

- [ ] Verify: scroll the page — chapters should load as they approach the viewport. Check browser DevTools Network tab to confirm lazy fetch pattern.
- [ ] **Commit:** `feat(reader): P5.3 — IntersectionObserver preload wiring`

## Task P5.5 — IndexedDB persistence for sha/etag

- [ ] Add IndexedDB persistence to `ChapterSection.svelte`:

1. After a successful `fetchCurrentEt` returns `{ status: 'fetched', sha, etag }`, persist `sha` and `etag` to IndexedDB keyed by chapter slug.
2. On component mount, check IndexedDB for a previously persisted `etag`. If found, pass it to `fetchCurrentEt` as `opts.etag` for a conditional 304 request.
3. Use the `idb-keyval` pattern (or a small inline wrapper) — no need for a full IndexedDB library. Store in a database named `bigbook` with a store named `chapter-meta`.

Keep the IndexedDB logic in a small helper: `src/lib/reader/idb.ts`:

```ts
export async function getChapterMeta(slug: string): Promise<{ sha: string; etag: string } | null>
export async function setChapterMeta(
  slug: string,
  meta: { sha: string; etag: string },
): Promise<void>
```

- [ ] Verify: load a chapter, check IndexedDB in DevTools Application tab (sha + etag stored). Reload page — the fetch should send `If-None-Match` and get 304 if unchanged.
- [ ] **Commit:** `feat(reader): P5.5 — IndexedDB persistence for sha/etag`

## Task P5.6 — Visibility-change refresh + return-visit scroll

- [ ] Add `visibilitychange` listener in `index.astro` page script:

1. When tab becomes `visible` after >2 minutes hidden, iterate all chapters currently in the preload buffer (or all loaded chapters), re-fire `fetchCurrentEt` with the stored `etag`.
2. 304 = no-op. 200 = re-parse, re-render the chapter, update IndexedDB.

- [ ] Wire return-visit scroll:

1. On page load, read `getLastParaId()` from `local-state.ts`.
2. If a value exists, scroll to `#<paraId>` after the skeleton renders (the skeleton rows have IDs from the manifest, so the scroll target exists immediately).
3. As the user scrolls, periodically update `setLastParaId()` with the first visible paragraph's `para-id`. Use the preload observer or a separate IntersectionObserver on paragraph rows.

- [ ] Wire prev/next chapter arrows in TopBar:

1. `#prev-chapter` and `#next-chapter` buttons scroll to the previous/next chapter section relative to `readerState.currentChapter`.
2. Force-fetch the target chapter if not already loaded.

- [ ] Verify: hide tab for >2 min, return — check that conditional refresh fires (Network tab). Navigate to a paragraph, close tab, reopen — should scroll back to the same position. Test prev/next arrows.
- [ ] **Commit:** `feat(reader): P5.6 — visibility-change refresh + return-visit scroll + nav arrows`
