# v1.1-content P2 ŌĆö Reader adaptation design

**Date:** 2026-04-20
**Milestone:** `v1.1-content`
**Phase:** P2 (reader adaptation)
**Author:** Plantin
**Status:** Approved by PO, pending implementation plan

## Problem

P1 (content bootstrap generator, PR #40) replaced the 16-section mock content tree with 68 canonical sections per language and introduced a new authoritative manifest at `src/content/manifest.json` carrying a `group` field per section.

The reader UI was built for the 16-section v1 shape. As a result:

1. `src/components/TocOverlay.svelte` hardcodes old mock slugs (`'eessonad'`, `'arsti-arvamus'`, `'lisad'`, `'cover'`, `'index'`) in its `FRONT_MATTER` / `APPENDICES` sets. None of those slugs exist in the new manifest, so the `groupLabel(slug)` function falls through for every section and the overlay renders all 68 items under a single "Chapters" heading with no visual grouping.
2. `tests/components/TocOverlay.test.ts` carries 2 failing assertions that hardcode `'ch01-billi-lugu'` and the 3-group shape.
3. Nine Playwright specs across 5 files (`tests/e2e/reader.spec.ts` + `tests/e2e/editor-*.spec.ts`) are `.skip`ped ŌĆö all use old-shape paraIds like `ch01-billi-lugu-p001` which no longer exist.

## Goal

Restore TOC grouping and unskip the nine Playwright specs by adapting the reader to the new manifest shape. Nothing more.

## Non-goals (explicit)

- **No backward compatibility.** No redirect map from old slugs to new, no paraId translation layer, no old-slug aliases, no URL-shape grace period. v1.0.0 shipped 3 days before v1.1.0-preview; the mock URLs are not load-bearing.
- **No fallbacks.** If a section has no `group` field, that is a data invariant violation and should fail loudly, not silently default to "Chapters".
- **No new TOC features.** The `stories` group is 42 entries; the existing 70vh scroll panel accommodates that. No collapsible groups, no search, no per-group jump nav. Flat list, four headings.
- **No global audit of the 8 reader-importing components.** The audit already happened during this design: a repo-wide grep for old mock slugs found only `TocOverlay.svelte` (code) and `src/pages/404.astro` (unrelated ŌĆö Jekyll-era URL prefixes that route to the archive). Content-file prose matches on bare word "lisad" are Estonian for "appendices" and are editorial, out of scope.

## Architecture

Data flow:

```
src/content/manifest.json
    |
    | (generated by scripts/bootstrap-content/emit-wrapper.ts)
    v
src/lib/content/manifest.ts   <-- projects manifest.sections into CHAPTERS
    |
    | CHAPTERS: readonly ChapterManifest[]
    v
src/components/TocOverlay.svelte   <-- groups by ch.group, renders 4 sections
```

### Change 1 ŌĆö `src/lib/content/manifest.ts`

Extend the `ChapterManifest` type and the `CHAPTERS` projection to carry `group`:

```ts
export type ChapterGroup = 'front-matter' | 'chapters' | 'stories' | 'appendices'

export type ChapterManifest = {
  slug: string
  group: ChapterGroup
  title: { en: string; et: string }
  paraIds: readonly string[]
}

export const CHAPTERS: readonly ChapterManifest[] = manifest.sections.map((s) => ({
  slug: s.canonicalSlug,
  group: s.group,
  title: s.title,
  paraIds: s.paraIds,
}))
```

The file is regenerated by `scripts/bootstrap-content/emit-wrapper.ts`. The generator module and any golden fixtures for it must emit the new shape.

### Change 2 ŌĆö `src/components/TocOverlay.svelte`

Delete:

- `FRONT_MATTER` slug set
- `APPENDICES` slug set
- `groupLabel(slug: string)` helper

Replace with a display-order and label table keyed on `ChapterGroup`:

```ts
const GROUP_ORDER: readonly ChapterGroup[] = ['front-matter', 'chapters', 'stories', 'appendices']
const GROUP_LABEL: Record<ChapterGroup, string> = {
  'front-matter': 'Front matter',
  chapters: 'Chapters',
  stories: 'Stories',
  appendices: 'Appendices',
}
```

The `groups` derived value iterates over `GROUP_ORDER`, filtering `chapters` by `ch.group === g`. The `Group.label` union type narrows to `string` (or the four literals if we keep the tighter typing ŌĆö no functional difference).

### Change 3 ŌĆö test rewrites

**`tests/components/TocOverlay.test.ts`** ŌĆö rewrite fixtures to use canonical slugs (`ch01`, `s01`, `fw1`, `a-i`, ŌĆ”) with explicit `group` values. Assert the four group headings render in order; assert a section from each group is placed under its correct heading.

**`tests/e2e/reader.spec.ts`** ŌĆö rewrite paraId fixtures from `ch01-billi-lugu-p001` shape to `ch01-p001` shape. Unskip all specs.

**`tests/e2e/editor-happy-path.spec.ts`**, **`editor-conflict.spec.ts`**, **`editor-auth-retry.spec.ts`**, **`editor-preflight-reject.spec.ts`** ŌĆö rewrite the shared fixtures module `tests/e2e/fixtures/editor-e2e.ts` (`makeBilingualChapter` likely hardcodes the old slug shape) and any per-spec paraId literals. Unskip all specs.

## Testing strategy

- **Component:** Vitest ŌĆö the rewritten `TocOverlay.test.ts` covers group plumbing + rendering + ordering. The generator golden fixtures cover the wrapper shape.
- **E2E:** Playwright matrix (chromium + firefox + webkit) on push to main, chromium only on PR. All 9 specs must pass un-skipped before merge.
- **Manual:** PO visual check that the overlay shows four group headings in order with the expected entries under each.

## Risks

- **The shared e2e fixtures module** (`tests/e2e/fixtures/editor-e2e.ts`) may have more hardcoded assumptions than a paraId-shape swap. If so, update in place ŌĆö still no fallbacks. Budget half a day for this if the fixture surface is larger than expected.
- **`stories` group scrolling on mobile.** 42 entries in the 80vh mobile panel. If PO flags it as UX-unacceptable during manual check, a collapsible-group feature becomes a separate ticket ŌĆö not a rework of P2.

## Success criteria

1. `CHAPTERS[i].group` is populated and typed as `ChapterGroup` for all 68 entries.
2. TocOverlay renders exactly four group headings in the order: Front matter, Chapters, Stories, Appendices. Each of the 68 sections appears exactly once under its correct heading.
3. `tests/components/TocOverlay.test.ts` passes (0 failures).
4. 9 previously-skipped Playwright specs pass un-skipped on chromium locally and on the full matrix in CI.
5. Full test suite passes: Vitest (Ōēł 330 + re-enabled specs, all green) and Playwright on the full matrix.
6. Zero net new `// TODO`, `// FIXME`, `// @ts-expect-error`, or `.skip` markers introduced.
